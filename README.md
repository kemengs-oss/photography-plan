# research-photography-plan

**摄影方案规划(爆款研究驱动版)** — 一个 Mira / Claude Skill。

给定**拍摄地点**和/或**主题**,先研究全网高热门的拍照攻略与样片,提炼出"被流量验证过的出片公式"(机位 / 姿势 / 穿搭 / 调色 / 文案),再结合用户的器材与个性化需求,生成一份**专业、可执行**的摄影方案,最终交付为一份**可翻页的 HTML PPT**。

## 核心理念:研究先行(Research-First)

不凭空讲摄影理论,而是**先看全网爆款,再做方案**。四层架构:

1. **意图识别** — 解析地点 / 主题 / 对象 / 时间 / 风格 / 器材
2. **爆款研究** — 多平台搜索高热门攻略,进原文交叉验证,提炼出片规律
3. **创意整合** — 爆款规律 + 用户个性化需求,给"复刻 + 差异化升级"
4. **可视化交付** — 整理为可翻页 HTML PPT,机位 Demo 页「参考图 + 极简线稿」并列

> **生成前确认**:研究完成后先在聊天里给出「爆款洞察 + 关键假设」,与用户对齐后再生成 PPT;这两块**不进 PPT**。

## 交付物特性(HTML PPT)

- **真·翻页**:一次只显示一页(方向键 / 空格 / PageUp·Down / 点击左右半屏);底部翻页控制条 + 全屏。
- **固定画布**:1280×720 标准 16:9,`transform` 等比缩放居中。
- **每页一屏不溢出**:内容超了就拆页,绝不截断或出现页内滚动。
- **版面均衡铺满**:内容上下左右均衡分布(`flex` 撑满 + `space-evenly` + 表格 `height:100%`),既不"太狭长"也不简单粗暴居中。
- **可模仿性**:每个机位/姿势 Demo 左右并列「真实参考图 + 极简几何线稿」,附姿势备注与适配妆容。

## PPT 结构(三阶段)

📋 **拍前准备** → 📸 **拍摄中** → 🎨 **后期**,每页带阶段角标,按决策合并相关内容:

1. 封面 · 2. 目录
3. 主题与氛围 · 4. 场景与时段 · 5. 服装与造型(含道具) · 6. 风险与备选
7. 机位造型 Demo(每页 ≤2 个) · 8. 拍摄流程 & Shot List(合并表)
9. 后期参考

## 使用方式

当用户提到「摄影方案 / 拍摄计划 / 旅拍 / 街拍 / 婚纱拍摄 / 人像拍摄 / 出片攻略」等需求时触发。即使用户只给了主题(如"想拍婚纱照")或只给了地点(如"去鼓楼大街拍照"),也会主动做缺口补全、推荐候选方向后再生成方案。

`examples/gulangyu-wedding-demo.html` 可直接用浏览器打开预览,作为版式与结构的参考样片——生成新方案时参照其排版,但内容须按当前地点/主题重做,不照搬。

## 适用题材

人像为默认骨架;美食 / 产品 / 宠物 / 风光等非人像主题会自动调整模块(姿势妆容 → 布景陈列 / 主体摆放 / 俯拍机位)。
